# Route business questions through runtime-local semantic query tools

Business questions are resolved through one shared executable semantic-query module, exposed to the model by a runtime-local Tool in Claude and Eve. The model may propose only catalog-backed terms; the module deterministically returns clarification, unsupported, or one certified query plan, and the selected runtime uses its own MCP Client to execute that plan against Toolbox. The shared capability activation separately publishes `semanticExecutionTools` and `modelSurface.visibleTools/hiddenTools`; runtime adapters translate that policy into their framework mechanisms, while compatibility aliases are not new facts of record. Business Toolbox Tools remain executable but are hidden from the model so they cannot bypass semantic resolution; platform-observability Tools may remain directly visible. This preserves runtime-owned MCP clients and the existing Agent run lifecycle while avoiding a new proxy process, a pre-`runAgent` intent branch, and arbitrary NL2SQL. This decision supersedes ADR 0018's single `allowedTools` activation surface but retains its Capability Pack and runtime ownership decisions.
